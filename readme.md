---

## Выводы

1. Схема Горнера быстрее прямого метода в ~19–100 раз благодаря линейной сложности O(n) вместо квадратичной O(n²)
2. Алгоритм Карацубы начинает выигрывать при размере многочлена N > 256. На малых N классическое умножение может быть наравне или даже чуть быстрее из-за накладных расходов на рекурсию
3. В Python рекурсивные вызовы дороже, чем в компилируемых языках, поэтому в коде установлен порог переключения на классический метод при N ≤ 32
